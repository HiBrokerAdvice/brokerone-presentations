PART 1 / LE PROBLÈME

Les courtiers ne font pas d'audit systématique.

Un client peut être chez le courtier depuis 10 ans. Entre-temps : des enfants, une maison, un changement de métier, une création de société. Mais personne ne revérifie son portefeuille.

⚠

Lacunes de couverture

Le client n'a pas d'assurance incapacité de travail. Il est indépendant. Un accident = plus de revenus. Le courtier est responsable de ne pas avoir détecté cette lacune.

💰

Doublons de couverture

Le client paie 2x l'assistance voyage : une dans son auto, une dans sa carte de crédit. Il surpaie. Il le découvre. Il est mécontent. Il change de courtier.

📈

Cross-sell manqué

Le client a un patrimoine immobilier mais aucune assurance propriétaire non-occupant. C'est du chiffre d'affaires que le courtier laisse sur la table.

⏳

Pas le temps

Un audit complet prend 45min-1h par client. Avec 500 clients, c'est 500 heures. Aucun courtier ne le fait manuellement.

⚖

Responsabilité juridique

Le courtier a un devoir de conseil. S'il ne détecte pas une lacune et que le client subit un sinistre non couvert, le courtier peut être tenu responsable.

😬

Clients insatisfaits

Un client qui découvre qu'il paie trop ou qu'il est mal couvert perd confiance. Le courtier perd le client et le bouche-à-oreille négatif commence.

PART 2 / LA SOLUTION

Comment Checkup fonctionne.

Checkup extrait les contrats du CRM, les analyse sur 17 axes, compare contre le référentiel IPID, et produit un score + des recommandations priorisées.

LE FLUX

5 étapes, 30 secondes.

1

Extraction CRM

Checkup se connecte au CRM du courtier (Brio, BrokerCloud) et récupère tous les contrats du client : branche, garanties, montants, dates.

2

Analyse 17 axes

Claude Sonnet analyse le portefeuille sur 8-9 axes Privé (mobilité, habitation, famille, santé, revenus, épargne, voyage, cyber, succession) + 9 axes Pro (RC Pro, locaux, personnel, flotte, BI, cyber, transport, homme-clé, crédit).

3

Comparaison IPID

Chaque contrat est mappé contre le référentiel IPID (218 garanties). Résultat : présent / absent / sous-couvert / doublon pour chaque garantie.

4

Score + Priorités

Score global 0-100. Score par axe. Classification des lacunes : Critique (action immédiate), Important (rapidement), Recommandé (quand possible). Top 3 priorités identifiées.

5

Rapport + Action

Génération d'un rapport HTML (in-app) + DOCX (téléchargeable). Pour les lacunes critiques : création automatique d'un dossier Augmented.

PART 4 / RÉFÉRENTIEL IPID

218 garanties référencées.

L'IPID (Insurance Product Information Document) est le document européen standardisé qui décrit les garanties de chaque produit d'assurance. Checkup utilise ce référentiel pour évaluer chaque contrat.

LE MAPPING IPID

Comment ça marche.

IPID

Base de données structurée

218 garanties classées par branche, sous-branche, et type. Chaque garantie a un code unique, une description, et un niveau d'importance (obligatoire / recommandé / optionnel).

IPID

Mapping automatique

Claude Sonnet analyse chaque contrat et mappe les garanties mentionnées aux codes IPID. "Assistance véhicule" dans le contrat → IPID-AUTO-007 (assistance dépannage).

IPID

4 statuts par garantie

Chaque garantie IPID est classée : Présent (couvert), Absent (non couvert), Sous-couvert (montant insuffisant), Doublon (couvert 2+ fois).

IPID

Pondération par profil

L'importance d'une garantie dépend du profil client. "Perte d'exploitation" est critique pour un indépendant avec 10 employés, optionnel pour un freelance solo.

Détection concrète.

✗IPID-HAB-003 Dégâts des eaux — Absent

✓IPID-HAB-005 Catastrophes naturelles — Présent

PRIORITÉS

Classification des lacunes.

CRITIQUE

Action immédiate requise

Risque majeur pour le client ou responsabilité juridique pour le courtier. Le courtier doit contacter le client dans les 48h.

Ex : Indépendant sans RC Pro, famille sans hospitalisation, entreprise sans BI
IMPORTANT

A traiter rapidement

Lacune significative mais pas de risque immédiat. A intégrer dans le prochain RDV client.

Ex : Pas de protection juridique, homme-clé non couvert, sous-couverture dégâts des eaux
RECOMMANDÉ

Quand c'est possible

Amélioration possible mais pas urgente. Mentionner lors d'un prochain contact.

Ex : Cyber privé, doublons assistance voyage, optimisation épargne pension

PART 7 / RAPPORTS

2 formats, 1 objectif.

Chaque audit génère un rapport complet en 2 formats : HTML interactif dans l'app, et DOCX téléchargeable pour les RDV clients.

🌐

Rapport HTML (in-app)

Interactif, scores visuels par axe, filtrage par priorité, drill-down sur chaque garantie. Accessible depuis le dashboard.

📄

Rapport DOCX (export)

Téléchargeable, imprimable, brandé avec le logo du courtier. Parfait pour les RDV clients et la traçabilité du devoir de conseil.

CONTENU DU RAPPORT

Ce qu'il contient.

1

Résumé exécutif

Score global, nombre de lacunes par priorité, phrase clé ("Ce portefeuille présente 5 lacunes critiques nécessitant une action immédiate").

2

Score par axe

Barre de progression pour chaque axe analysé. Code couleur : rouge (<40), orange (40-60), jaune (60-80), vert (>80).

3

Détail des lacunes

Pour chaque lacune : description, impact, urgence, action recommandée, prime estimée.

4

Recommandations priorisées

Top 3 priorités avec argumentaire. Le courtier peut directement utiliser ce texte en RDV client.

5

Carte de couverture existante

Vue complète de ce que le client a déjà : contrats, garanties, montants, échéances. Sert aussi de base pour le renouvellement.
